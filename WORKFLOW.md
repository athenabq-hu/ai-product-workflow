# 完整工作流详解

> 本文档是 `README.md` 的详细展开，包含每个阶段的完整操作步骤与技巧。

---

## 工作流总览

```
第一阶段 → 第二阶段 → 第三阶段 → 第四阶段 → 第五阶段 → 第六阶段
（行业研究）  （内容整理）  （产品定义）  （UI设计）   （动效）    （代码）
  1-2天        半天         1-2天        2-3天       1-2天      3-5天
```

**总计**：一个 AI 产品原型，约 **2-3 周**完成

---

## 第一阶段：行业研究

### 目标
搞清楚：市场有多大、竞争对手是谁、用户真正需要什么

### 准备工作
1. 明确产品方向（"AI + 什么赛道"）
2. 收集初步关键词列表
3. 确定目标用户画像（至少 3 类核心用户）

### 研究框架
```
1. 市场规模分析
   - TAM / SAM / SOM
   - 年增长率（CAGR）
   - 驱动因素

2. 竞争格局
   - 直接竞品（做一模一样事情的）
   - 间接竞品（解决同一个问题的不同方式）
   - 潜在进入者（大厂动向）

3. 用户研究
   - 社交媒体监控（小红书/微博/知乎/App Store）
   - 论坛和社区（V2EX/Reddit/Product Hunt）
   - 现有产品评论（挖掘真实痛点）

4. 技术评估
   - 现有 AI 能力边界
   - 成本结构
   - 监管风险
```

### 输出标准
- 市场数据要有来源（研报/官方发布/第三方统计）
- 竞品对比至少 5 个维度
- 用户痛点要有真实引用（而不是猜测）

## 第二阶段：内容整理

### 目标
将研究信息提炼成有说服力的洞察，用于内部对齐和融资准备

### PPT 制作规范
```
尺寸：16:9（1920×1080px）
主色：#1A56DB（品牌蓝）
强调色：#F59E0B（信息黄）
背景：纯白 #FFFFFF
字体：标题 Arial Bold 36pt，正文 Arial 18pt
```

### 麦肯锡金字塔原则
- **结论先行**：每页只说一个核心观点
- **逻辑支撑**：每个观点有 2-3 个数据点支撑
- **数据说话**：能用图表的不要用文字
- **行动导向**：每部分结尾要有明确的下一步

## 第三阶段：产品定义

### PRD 撰写要点

**产品定位公式**：
> [产品名称] 是一款面向 [目标用户] 的 [产品类别]，帮助他们在 [使用场景] 下完成 [核心任务]，不同于 [竞品]，我们的优势是 [差异化]。

**用户故事格式**：
```
作为一个 [用户类型]，
我希望 [完成某个任务]，
以便 [达成某个目标]。

验收标准：
- [可衡量的结果 1]
- [可衡量的结果 2]
```

**功能分层（L1-L5）**：
- **L1（基础）**：解决"有没有"的问题，上线前必须完成
- **L2（进阶）**：解决"好不好用"，V2 完成
- **L3（高级）**：解决"用得爽"，V3 完成
- **L4（增强）**：差异化竞争优势，中期规划
- **L5（探索）**：前沿技术验证，长期储备

## 第四阶段：UI/UX 设计

### 设计流程
```
1. 信息架构（IA）
   - 页面清单
   - 导航结构
   - 内容层级

2. 关键流程（User Flow）
   - 核心路径走通
   - 分支路径处理
   - 异常路径兜底

3. 低保真原型
   - 手绘线框图
   - 确认布局逻辑

4. 高保真设计
   - 视觉设计
   - 组件状态
   - 标注切图
```

### AI 产品设计要点
- **AI 思考中**：要有明确的视觉反馈，不能让用户面对空白屏幕
- **生成结果**：要有预览、有编辑、有反馈
- **容错设计**：AI 输出不稳定，要预留"重新生成"入口
- **个性化**：记住用户偏好，减少重复操作

## 第五阶段：动效设计

### AI 产品动效原则
1. **AI 思考动效**：不能只是 spinner，用有意义的微动画暗示 AI 在工作
2. **生成完成动效**：结果出现要有仪式感，增强用户信任
3. **错误处理动效**：出错了要给用户"希望"而不是焦虑

### 推荐动效工具
- **ProtoPie**：免费，支持复杂交互，导出视频
- **Framer**：可直接做可交互原型
- **CSS Animations**：代码级，可直接嵌入 Web 应用

## 第六阶段：代码开发

### 技术选型决策树
```
需求复杂度低 + 需要快速验证
→ Next.js + Tailwind + AI API

需求复杂 + 需要 SEO
→ Next.js + React + AI API

需要实时交互
→ Next.js + WebSocket + AI API

需要离线能力
→ Tauri / Electron + AI API

移动端优先
→ React Native + AI API
```

### AI API 集成检查清单
- [ ] API Key 安全存储（不在前端暴露）
- [ ] 请求超时处理（AI 响应可能慢）
- [ ] 错误状态 UI（网络错误/限流/内容违规）
- [ ] 加载状态动效（AI 处理中）
- [ ] 结果缓存（避免重复请求）
- [ ] 内容审核（防止用户输入恶意内容）
